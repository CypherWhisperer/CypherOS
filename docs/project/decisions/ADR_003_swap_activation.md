# ADR-003: Swap Activation

**Date:** 2026-04-15
**Status:** Accepted
**Deciders:** CypherWhisperer
**Related:** [INC-2026-04-15-001](https://claude.ai/incidents/INC-2026-04-15-001.md) _(This was a direct consequence of the OOM incident. The swapfile existed on disk but NixOS had never been told about it.)_

---

## Context

The `@swap` BTRFS subvolume was created during the initial disk setup and mounted at `/swap`. A 10GB swapfile existed on disk — _created with `fallocate -l 10G` and initialized with `mkswap` during the install_. The `@swap` subvolume had Copy-on-Write disabled (`chattr +C`) as required.

However, `hardware-configuration.nix` had `swapDevices = [ ]` — empty. The swapfile existed on disk, but **NixOS had never been told about it and was not activating it at boot.** The machine was running with no effective swap at all.

This became apparent during the OOM incident ([INC-2026-04-15-001](../../development/incidents/INC_2026_04_15_001.md)): a `nixos-rebuild switch` triggered source builds of memory-hungry packages (_`terraform`, `vault`, `n8n`_), the kernel OOM killer fired, and the terminal crashed. With an activated swapfile, the worst-case outcome would have been a slow build rather than a crash.

Activating the swap was addressed as part of the OOM incident response.

---

## Decision

Declare `swapDevices` in `configuration.nix` pointing to `/swap/swapfile`. This makes NixOS activate the swapfile at boot via systemd.

---

## Reasoning

The fix is straightforward — _one declaration in `configuration.nix`_:

```nix
# ─────────────────────────────────────────────────────────────────────────────
# SWAP
# ─────────────────────────────────────────────────────────────────────────────
# The @swap BTRFS subvolume is mounted at /swap (declared in hardware-configuration.nix).
# The swapfile was created during install (fallocate -l 10G + mkswap).
# Declaring it here makes NixOS activate it at boot via systemd.
#
# BTRFS swapfiles have one hard requirement: the file must live on a subvolume
# with Copy-on-Write disabled (chattr +C). This is handled in the disk setup
# bash script on @swap — so this declaration is safe.
swapDevices = [
  {
    device = "/swap/swapfile";
    # size is informational here — the file is already sized at 10G on disk.
    # NixOS will not resize it; it just activates it.
  }
];
```

**Why `configuration.nix` and not `hardware-configuration.nix`:**

`hardware-configuration.nix` is auto-generated by `nixos-generate-config` and gets overwritten every time that command is run (_e.g. after hardware changes_). Anything you manually declare in it is at risk of being wiped. `swapDevices` belongs in `configuration.nix` even though it feels like hardware config — _it is a deliberate system configuration choice, not a detected hardware fact_.

**BTRFS swapfile hard requirement:**

BTRFS swapfiles require the containing subvolume to have Copy-on-Write disabled. The kernel hard-refuses to activate swap on a CoW-enabled BTRFS file. The `@swap` subvolume had `chattr +C` applied during the initial disk setup script. This declaration is therefore safe.

**Verification after a rebuild:**
```bash
# Should show the 10G swapfile as active
swapon --show

# Should show swap in use (Size column non-zero)
free -h
```

---

## Alternatives Considered

### Declare in `hardware-configuration.nix`

This is where `swapDevices` is auto-detected and placed if a swap device exists at the time `nixos-generate-config` runs. The problem is that `hardware-configuration.nix` is machine-generated and gets overwritten. Any manual addition there is fragile. `configuration.nix` is the correct place for anything you manually declare.

### Use a swap partition instead of a swapfile

A dedicated swap partition is simpler (_no BTRFS CoW caveat, no `chattr +C` requirement_) but requires pre-allocating space at partition time. BTRFS subvolumes are explicitly designed to avoid pre-allocation — adding a swap partition would mean leaving space on the table in the BTRFS pool or repartitioning. The swapfile approach preserves the _"one BTRFS partition for everything"_ design.

---

## Consequences

**Positive:**

- NixOS activates swap at boot. The machine now has 10GB of disk swap as a fallback behind ZRAM.
- Combined with ZRAM ([ADR-004](./ADR_004_zram_setup.md)), OOM scenarios now have two layers of cushion before the kernel fires the OOM killer.

**Negative / Trade-offs:**

- Swap on a spinning disk (_or even SSD_) is slow. It is explicitly the last resort — _ZRAM absorbs pressure first, disk swap only if ZRAM fills._

**Neutral / Operational:**

- The swapfile size (_10GB_) is fixed by the disk setup script. NixOS reads the `device` path and activates whatever is there. Resizing requires recreating the swapfile manually.
- This declaration must survive any future `nixos-generate-config` run — the generated file does not know about the manually declared swapfile.
